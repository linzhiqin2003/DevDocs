# Web 服务架构：Nginx + Gunicorn + Django

## 一句话说清楚三者关系

用户的浏览器发出一个请求，到你的 Django 代码处理它，中间要经过三层：

```
用户浏览器
    │
    │  HTTPS 请求
    ▼
┌─────────────────────────────────────────────┐
│  Nginx（反向代理）                            │
│  - 接收所有外部请求                            │
│  - 静态文件直接返回（CSS/JS/图片）              │
│  - 动态请求转发给 Gunicorn                     │
│  - 处理 SSL 证书、负载均衡                     │
│  - 如果 Gunicorn 超时 → 返回 504              │
└──────────────────┬──────────────────────────┘
                   │
                   │  HTTP 转发（内网通信）
                   ▼
┌─────────────────────────────────────────────┐
│  Gunicorn（应用服务器 / 进程管理器）            │
│  - 启动多个 worker 进程                       │
│  - 每个 worker 跑一份完整的 Django             │
│  - 并行处理多个请求                            │
│  - worker 挂了自动重启                        │
└──────────────────┬──────────────────────────┘
                   │
                   │  调用 Django WSGI 接口
                   ▼
┌─────────────────────────────────────────────┐
│  Django（Web 框架）                           │
│  - 路由匹配（URL → View）                     │
│  - 执行业务逻辑（查数据库、调 API）             │
│  - 返回响应（JSON / HTML）                    │
└─────────────────────────────────────────────┘
```

## 类比

把它想象成一家餐厅：

| 角色 | 对应组件 | 职责 |
|------|---------|------|
| 门口迎宾 | **Nginx** | 接待客人，简单需求直接处理（倒杯水），复杂的安排到座位 |
| 大堂经理 | **Gunicorn** | 管理所有服务员（worker），确保每桌都有人服务 |
| 厨师 | **Django** | 真正做菜的人，处理具体的业务逻辑 |

门口迎宾不会做菜，厨师不需要招呼客人，各司其职。

## 一个真实请求的完整旅程

用户在浏览器访问 `https://your-app.com/api/reports/` 时，发生了什么：

1. **DNS 解析**：域名 → 服务器 IP
2. **Nginx 接收**：443 端口收到 HTTPS 请求，SSL 解密
3. **Nginx 判断**：这是 `/api/` 开头，不是静态文件，转发给 Gunicorn
4. **Gunicorn 分配**：找一个空闲的 worker 来处理
5. **Django 处理**：middleware → URL 路由 → view 函数 → 查数据库 → 返回 JSON
6. **原路返回**：Django → Gunicorn → Nginx → 用户浏览器

整个过程通常在几十毫秒内完成。但如果第 5 步要做一个耗时 2 分钟的 PDF 索引，问题就来了——详见各组件的文章。

## 本地开发 vs 生产环境

| | 本地开发 | 生产环境 |
|--|---------|---------|
| 前端请求到达 | 直接访问 `localhost:8000` | 经过 Nginx 反向代理 |
| 应用服务器 | `manage.py runserver`（单进程） | Gunicorn（多 worker） |
| 静态文件 | Django 自己伺服 | Nginx 直接返回 |
| HTTPS | 没有 | Nginx 处理 SSL |
| 并发能力 | 基本没有 | 取决于 worker 数量 |

这就是为什么本地开发一切正常，上了生产可能会遇到各种超时、并发问题。
